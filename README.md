🏆 30-Day Champions Training Plan — RealPage AI Enablement
We are launching a structured 30-day programme to train and certify 30 pod-level champions across RealPage. 4 architects will lead the training, covering GitHub workflows, QA STLC phases, AI agents, and Claude usage. Here's the full plan 👇
📌 Overview
4 architects will each mentor ~7–8 pod champions
30 champions identified and nominated by pod-level QA Managers
Pre-phase (10 days) + 30-day training plan = ~40 days total
All 4 focus areas covered by every architect across all champions
🎯 4 Focus Areas (all architects cover all 4)
🟠 Test case generation & STLC coverage
🟣 PR reviews & automation scripts
🟢 GitHub workflows & runner strategy
🔵 Code validation, report creation & analysis
📅 Week-by-Week Breakdown
Pre-phase Days 1–10 · Architect preparation & champion identification
4 architects trained on GitHub Actions, workflow YAML and QA STLC phase mapping
Runner types defined — hosted vs self-hosted, configuration and sizing
Architects define champion selection criteria and share with QA Managers
Pod QA Managers nominate champions — target 30 confirmed and rostered
Champions informed of plan, dates, and expectations
✅ Outcome: 4 trained architects + 30 identified champions ready to begin

Week 1 Days 11–17 · Onboarding and foundations
Kickoff session for all 30 champions — goals, plan walkthrough, success criteria
realpage-powerup setup verification and Claude access confirmed for all champions
Intro to all AI agents — user story evaluation, test case generation, PR analyzer
Overview of GitHub workflow structure and QA STLC alignment
All 4 focus areas introduced with live demos and Q&A
✅ Outcome: All 30 champions onboarded, tools ready, baseline knowledge established

Week 2 Days 18–24 · Hands-on training — pod group sessions
Each architect runs intensive sessions with their ~7–8 assigned champions
Test case generation using Claude — writing, reviewing, and refining TCs
Automating PR pre-reviews with AI — hands-on walkthroughs on real PRs
Building GitHub workflows for unit and integration STLC phases
Mid-week cross-pod sync — champions share learnings, architects resolve blockers
✅ Outcome: Champions hands-on with all 4 focus areas, initial workflows drafted

Week 3 Days 25–31 · Deep dives and applied practice
Champions complete end-to-end GitHub workflow for their pod independently
Full STLC pipeline built — from unit tests through to release gate
Automation scripts reviewed, refined, and committed to repo
Cross-pod showcase — each champion presents their workflow to peers
Progress check — architects assess readiness for certification week
✅ Outcome: Full workflows operational, champions ready for independent execution

Week 4 Days 32–40 · Independent practice and certification
Champions deliver a complete QA workflow independently — no architect guidance
Live demonstration of all 4 focus areas to their assigned architect
Formal certification sign-off by architect on completion
Each champion runs their first training session with their own pod team
Continuous improvement and monthly feedback loop opened with architects
✅ Outcome: 30 certified champions — AI and GitHub workflow adoption cascading org-wide
✅ Champion Selection Criteria
Pod QA Managers should nominate champions based on the following criteria:
Criteria	What to look for	Priority
QA knowledge & activities	Has owned QA activities end-to-end across STLC phases — test planning, execution, defect management, and release validation	Must have
AI tooling & agent usage	Actively using RealPage AI agents for QA tasks — user story evaluation, test case generation, PR impact analysis	Must have
Claude proficiency	Well-versed in Claude usage — prompt crafting, iterating outputs, applying Claude to real QA scenarios beyond basic queries	Must have
Skills & agent framework	Familiar with realpage-powerup skills and agent structure — understands how agents are triggered and outputs consumed in QA	Must have
Pod influence & communication	Go-to person in the pod; can teach, document, and mentor peers on AI tooling and QA best practices	Important
Availability & commitment	Can dedicate time across the 30-day plan; has manager sign-off for participation	Important
GitHub workflows	Basic familiarity with GitHub Actions, workflow YAML, and CI/CD concepts — full expertise not required	Good to have
